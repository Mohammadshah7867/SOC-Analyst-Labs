# ARP Spoofing Detection with Wireshark

## Objective

Investigate network traffic to identify signs of ARP spoofing and determine whether a host is attempting to impersonate the network gateway.

## Tools Used

* Wireshark
* TryHackMe MITM Detection Lab

## Investigation Steps

### 1. Review ARP Traffic

I filtered for ARP traffic using:

`arp`

This allowed me to view all ARP requests and responses and establish a baseline of normal network behavior.

![ARP Requests](../arp.opcode%20==1.png)

*Figure 1: ARP requests displayed in Wireshark using the filter `arp.opcode == 1`.*

### 2. Examine ARP Requests

I filtered ARP requests using:

`arp.opcode == 1`

This displayed "who-has" requests generated by hosts attempting to resolve MAC addresses.

### 3. Examine ARP Responses

I filtered ARP responses using:

`arp.opcode == 2`

The results showed multiple ARP replies, including unsolicited responses that warranted further investigation.

![ARP Responses](arp_opcode_2.png)

### 4. Identify Gratuitous ARP Replies

I searched for gratuitous ARP traffic using:

`arp.isgratuitous`

These packets are often associated with ARP cache poisoning because they advertise IP-to-MAC mappings without a corresponding request.

### 5. Investigate Gateway Traffic

I focused on traffic associated with the gateway IP address (192.168.10.1) and observed repeated ARP replies claiming ownership of the gateway address.

### 6. Detect ARP Spoofing

Further analysis revealed that the MAC address `02:fe:fe:fe:55:55` repeatedly associated itself with the gateway IP address. This behavior is consistent with ARP spoofing.

### 7. Validate Duplicate IP-to-MAC Mappings

Using Wireshark's duplicate address detection filters, I confirmed that multiple MAC addresses claimed the same IP address, indicating a successful ARP poisoning attempt.

## Findings

* ARP spoofing activity was detected.
* The gateway IP address was impersonated.
* Attacker MAC address identified: `02:fe:fe:fe:55:55`.
* Duplicate IP-to-MAC mappings confirmed malicious activity.

## Lessons Learned

ARP spoofing attacks can be identified by monitoring gratuitous ARP traffic, duplicate IP-to-MAC mappings, and repeated ARP replies claiming ownership of critical network devices such as gateways.
