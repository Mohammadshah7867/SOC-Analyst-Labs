# Digital Forensics Investigation – FTK Imager & Autopsy

## Objective
Conduct a forensic examination of a USB flash drive image while preserving evidence integrity and documenting findings using industry-standard digital forensic tools.

---

## Scenario
This project was completed as part of a George Mason University Digital Forensics course. The investigation involved examining a forensic image of a USB flash drive associated with a fictional FBI investigation. The objective was to verify the integrity of the evidence, analyze the file system, recover relevant artifacts, and document the findings in a professional forensic report.

> **Note:** This is an educational project based on a fictional case scenario.

---

## Tools Used

- FTK Imager
- Autopsy
- Windows
- MD5 / SHA-1 Hash Verification

---

## Skills Demonstrated

- Digital Evidence Handling
- Evidence Integrity Verification
- MD5 & SHA-1 Hash Validation
- File System Analysis
- FAT16 File System Investigation
- Deleted File Recovery
- Metadata Analysis
- Digital Forensics Documentation
- Technical Report Writing

---

## Investigation Process

1. Verified forensic image integrity using MD5 and SHA-1 hashes.
2. Loaded the evidence image into Autopsy.
3. Examined the FAT16 file system.
4. Identified active and deleted files.
5. Analyzed Office documents, PDFs, HTML files, images, and embedded objects.
6. Documented timestamps, metadata, and recovered artifacts.
7. Produced a complete forensic examination report.

---

## Key Findings

- Successfully verified image integrity before analysis.
- Identified numerous active and deleted artifacts.
- Recovered deleted files and analyzed document metadata.
- Examined embedded objects and web artifacts.
- Documented all findings while maintaining forensic best practices.

---

## Report

A complete forensic report was created documenting the examination methodology, evidence handling procedures, analysis, and findings.
